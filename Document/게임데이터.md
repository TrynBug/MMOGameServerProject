# 게임데이터 개요
- 게임데이터는 게임내 로직을 위한 데이터로, 스테이지 데이터, 몬스터 데이터, 아이템 데이터 등이 있다.
- 서버, 클라에서 공유하여 같이 사용한다.
- 기획자는 .xlsx 파일에 데이터를 입력한다.
- 서버, 클라이언트는 .xlsx 파일의 데이터를 .csv로 변환한 것을 사용한다.
- GameDataGenerator C# 프로젝트가 .xlsx 파일을 .csv 파일로 변환하고, 데이터에 해당하는 코드파일(서버:c++, 클라:C#)을 생성한다.
- 서버에서는 GameDataLib c++ 프로젝트가 GameDataGenerator가 생성한 c++ 파일들을 빌드하여 하나의 라이브러리로 만든다.

# 게임 enum 개요
- 게임데이터 내에서 enum을 사용하기 위해 게임 enum을 따로 관리합니다.
- 기획자 및 프로그래머는 GameEnum.xlsx 파일에 enum을 입력합니다.
- GameDataGenerator C# 프로젝트가 GameEnum.xlsx 파일을 enum 코드파일로 변환합니다.
- 게임데이터는 GameEnum.xlsx 파일을 참조하여 enum값을 숫자값으로 변경하여 .csv로 생성합니다.

# GameDataGenerator 프로젝트
- .NET 8.0, C# 언어 버전 12.0 사용
- output은 exe 파일이다.
	- 출력 경로는 프로젝트경로/../OUTPUT/Tools 이다.
- 데이터파일 처리할때
	- 인자로 데이터파일명, 참조할 enum xlsx 파일 경로, 코드파일 출력위치, 데이터파일 출력위치를 입력받아서 데이터파일과 코드파일을 생성한다. 옵션에 따라 데이터파일만 생성하기, 코드파일만 생성하기, 둘다 생성하기를 할 수 있다.
- enum파일 처리할때
	- 인자로 enum xlsx파일명, 코드파일 출력위치를 입력받아서 enum이 선언된 c++ 파일과 c# 파일을 생성한다.
- 서버/클라 시작 시 모든게임데이터 한번에 로드하는 코드 만들때
	- 인자로 코드파일 위치를 전달받아, 해당 위치에 있는 모든 게임데이터 클래스의 인스턴스를 생성하는 GameDataManagerBase 파일을 생성한다.
- 인자의 상세 내용은 GameDataGenerator 프로젝트의 Program.cs 파일 최상단 주석을 확인하면 된다.

	- 코드파일 및 데이터파일에 입력될 필드명이다. 
	- 만약 필드명의 마지막에 숫자가 있다면 코드파일에서는 특별한 규칙이 생긴다:
		- 예를들면 이름이 Stat1, Stat2, Stat3 필드가 있다고 가정한다.
		Stat1, Stat2, Stat3 필드는 코드파일에 실제로 존재하고, 여기에 추가로 Stat# 필드를 순회할 수 있는 기능을 제공해야 한다.
		예를들면 GetStatCount() 함수는 3을 리턴해야 하고, GetStat(int index) 함수는 index가 0이면 Stat1을 리턴, index가 1이면 Stat2를 리턴, index가 2이면 Stat3을 리턴한다.
		이 함수들은 아래와 같이 순회할 때 사용한다:
		```cpp
		GameData_Monster* pMonsterData = GetGameDataMonster(1);
		for ( int i=0; i<pMonsterData->GetStatCount(); ++i)
		{
		  Stat stat = pMonsterData->GetStat(i);
		}
		```
		- 이런 규칙을 보았을 때, 필드명의 마지막에 숫자가 있다면 반드시 1부터 시작하여 1씩 증가해야 한다. 그리고 Stat# 필드들의 데이터타입은(5행의 값) 모두 같아야 한다. 
		- 필드명의 마지막에 숫자가 있는 필드는 여러개가 있을 수 있다. 예를들면 .xlsx 파일의 8행에 Stat1, StatValue1, Stat2, StatValue2, Stat3, StatValue3 와 같이 컬럼명이 올 수 있다.

	- 1개 행이 1개 데이터를 나타낸다.
	- 5행(데이터타입)이 "int", "string", "double", "bool" 일 경우 해당 타입의 데이터가 입력된다.

# GameDataLib 프로젝트
이 프로젝트는 서버에서 게임데이터를 쉽게 로드하고 사용할 수 있도록 기능을 제공하는 프로젝트입니다. 
- c++ 20
- output은 static library 파일이다.
	- 출력 경로는 $(SolutionDir)OUTPUT/$(Configuration)/
	
서버에서는 GameDataLib.h 를 include 하고, GameDataManager::LoadAllGameData(const std::string& csvPath) 함수로 모든 게임데이터를 로드합니다.  
```cpp
#include "GameDataLib.h"

const std::filesystem::path currentDir = std::filesystem::current_path();
const std::filesystem::path gameDataDir = currentDir.parent_path() / "GameData";
GameDataManager::LoadAllGameData(gameDataDir.string());
```

그리고 각각의 게임데이터를 사용할 때는 필요한 게임데이터 클래스를 직접 include 해야합니다.
예를들면 Monster 데이터를 사용하려한다면 이렇게 데이터를 얻습니다.
```cpp
#include "Generated/GameData_Monster.h"

const GameData_Monster* pData = GameDataTable_Monster::FindData(100);
```

게임데이터는 기본적으로 서버가 시작될 때 한번 로드되고, 그 뒤로는 절대로 수정되지 않습니다.  
데이터 사용자도 절대로 데이터를 수정해서는 안됩니다.
